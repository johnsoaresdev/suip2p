# Contribuindo

## Commits
Use Conventional Commits: `feat: ...`, `fix: ...`, `chore: ...`.

## Branches
- `main`: produção
- `develop`: integração
- feature branches: `feat/<nome>`

## PR Checklist
- [ ] Lint ok
- [ ] Testes passaram
- [ ] Documentação atualizada
- [ ] Para contratos: adicionar/atualizar arquivos em `deployments/` caso necessário
